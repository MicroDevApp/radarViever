# MapGlobe

Qt6/QML + OpenGL (C++) тайловая карта: панорамирование, зум, и плавное
сворачивание плоской карты в сферу планеты при отдалении. Рельеф (горы/
впадины/равнины) — hillshading по реальным данным высот, без отдельной
3D-геометрии (флэт-сетка, шейдинг в фрагментном шейдере).

## Сборка

Нужен Qt6 (Core, Gui, Qml, Quick, Network, OpenGL) и компилятор с C++17.

```bash
mkdir build && cd build
cmake .. -DCMAKE_PREFIX_PATH=/path/to/Qt/6.x.x/gcc_64
cmake --build . -j
./MapGlobe
```

Приложению нужен доступ в интернет (тайлы OSM + DEM качаются по HTTP(S) и
кешируются на диск).

## Архитектура

- `TileCache` — кеш одного слоя тайлов: память -> диск
  (`<AppData>/maps/<layer>/z/x/y.png`) -> сеть. Используется два экземпляра:
  `osm` (цветные тайлы) и `dem` (Mapzen Terrarium, высоты).
- `MapView` (`QQuickItem`) — состояние камеры (lon/lat/zoom), ввод
  (`panPixels`, `zoomBy`), на каждый кадр считает список видимых тайлов и
  матрицу `viewProj`, блендит камеру между "плоским top-down" и
  "орбитальным сферическим" режимами по `globeBlend()`.
- `MapRenderNode` (`QSGRenderNode`) — сырой OpenGL: одна общая unit-сетка
  17x17 вершин переиспользуется для всех тайлов (через uniform `uTileRect`),
  на тайл — два sampler'а (цвет + высота). Вершинный шейдер блендит
  flat/sphere позицию, фрагментный считает hillshading по градиенту высот.

## Рельеф без геометрии

Высоты (Terrarium: `height = R*256 + G + B/256 - 32768`) сэмплятся в 4
соседних точках DEM-текстуры, по разнице строится псевдо-нормаль и diffuse-
освещение, которым умножается цвет тайла. Сетка остаётся плоской — это
просто текстурный эффект, но привязанный к настоящим данным рельефа.

## Что точно нужно подстроить под себя

- `uReliefStrength` (MapRenderNode.cpp) и диапазон `mix(0.55, 1.25, diff)` —
  сила тени гор, подбирается визуально.
- `hFlat`, `dSphere`, `kGlobeMaxZoom` (MapView.cpp/.h) — на каком zoom
  карта становится плоской/сферой, и насколько камера далеко в каждом
  режиме.
- User-Agent в `MapView::MapView()` — впишите реальный контакт, это
  требование Tile Usage Policy OSM.

## Известные ограничения (осознанные, для демо-масштаба проекта)

- Тайл `tile.openstreetmap.org` — это сервер для лёгкого/тестового
  использования, не для продакшена/массовой нагрузки (см. их Tile Usage
  Policy). Для реального приложения — свой тайл-сервер или платный
  провайдер (MapTiler, Mapbox и т.п.), просто меняется URL-шаблон в
  `TileCache`.
- Чтение диск-кеша в `TileCache::tile()` происходит синхронно в момент
  `updatePaintNode` (sync-фаза, GUI-поток заблокирован, поэтому это
  безопасно по потокам, но при холодном диск-кеше может давать микро-
  лаги). При необходимости легко увести на отдельный поток.
- Видимые тайлы на высоком zoom считаются простым окном вокруг центра, без
  фрустум-куллинга и без LOD-пирамиды (тайл соседнего уровня как заглушка
  пока грузится текущий) — другими словами, при подгрузке тайл просто не
  рисуется, а не показывает blurred-превью.
- Блендинг камеры flat↔sphere — линейная интерполяция eye/center/up, а не
  честная непрерывная проекция (как в Google Maps/Cesium). Визуально
  выглядит нормально, но не математически "точно".
- Около полюсов (широта ±85° и выше) Mercator вырождается — широта жёстко
  ограничена `qBound(-85, ..., 85)`.
- Получившиеся швы на границах тайлов в hillshading (сосед недоступен при
  градиенте на краю текстуры) — мелкий визуальный артефакт, не влияет на
  общую читаемость рельефа.

Код не компилировался в этой среде (нет Qt6/дисплея в контейнере) — структура
и API использованы по актуальной памяти Qt6, но при сборке возможны мелкие
несовпадения сигнатур (особенно `QSGRenderNode::StateFlags`/`RenderingFlags`
могут отличаться по доступным enum-значениям в вашей минорной версии Qt6).
